# Overview

</header>

<Callout type="warning" emoji="⚠️">
This documentation is currently under development.
</Callout>

In this Getting Started track, you'll build and deploy a FastStore Storefront at the domain `https://<account-name>.vtex.app/`.

By the end of this track, you'll have a fully functional store using FastStore, seamlessly integrated with VTEX native solutions like [Headless CMS](https://www.faststore.dev/tutorials/cms/0).
Here's an example of the final result: [Starter Store](https://starter.vtex.app).
  
## 1. [FastStore Onboarding](/docs/getting-started/onboarding)

Start your project with the FastStore Onboarding app. It streamlines the process by setting up the essential store configurations, creating a Github repository for your store, and providing a live demo preview of your store.

## 2. [Setting Up the Project](/docs/getting-started/settingup-project)

Discover the core technologies that we will be using in this tutorial and set up all the necessary tools to implement your very first FastStore storefront and create your first storefront customization.

## 3. [Understanding the Project Structure](/docs/getting-started/project-structure)

Find out how to organize your FastStore project and get to know our recommended conventions for naming files and folders.

## 4. [FastStore CLI](/docs/getting-started/faststore-cli)
The FastStore Command Line Interface (CLI) is the main pre-configured tool for starting, building, and developing FastStore projects.
