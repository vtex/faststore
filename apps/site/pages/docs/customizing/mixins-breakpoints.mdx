## Breakpoints

To ease our screen breakpoint customization, we're using [Include-Media](https://eduardoboucas.github.io/include-media/#features), a Sass library for writing CSS media queries in an easy and maintainable way, using a natural and simplistic syntax. We've defined the 5 following screen breakpoints:

```scss
$breakpoints: (
  "phone":         320px,
  "phonemid":      375px,
  "tablet":        768px,
  "notebook":      1280px,
  "desktop":       1440px,
);
```

### Customizing Breakpoints

If you want to override this sizing system, follow the following steps:

1. Go to the `src` folder and create the `styles` folder inside it. You can run the following command to create it:

<Tabs items={['macOS and Linux', 'Windows']}>
  <Tab>
    ```bash copy
    mkdir src/styles
    ```
  </Tab>
  <Tab>
    ```bash copy
    mkdir src\styles
    ```
  </Tab>
</Tabs>

2. Inside the `styles` folder, create a new file called `custom-mixins.scss`

<Tabs items={['macOS and Linux', 'Windows']}>
  <Tab>
    ```bash copy
    touch src/styles/custom-mixins.scss
    ```
  </Tab>
  <Tab>
    ```bash copy
    touch src\styles\custom-mixins.scss
    ```
  </Tab>
</Tabs>

3. Copy and paste the following code snippet into the file:

```scss {8} filename="src/styles/custom-mixins.scss" copy
@import "@faststore/ui/utilities"

$breakpoints: (
  "phone":         320px,
  "phonemid":      375px,
  "tablet":        768px,
  "notebook":      1280px,
  "desktop":       1500px,
);
```

<Callout type="warning" emoji="⚠️">
  You need to declare all the variants (phone, phonemid, tablet, notebook and desktop).
</Callout>

4. Run `yarn dev` in the terminal to start the development server.

## Mixins

We use the [Sass `@mixin`](https://sass-lang.com/documentation/at-rules/mixin/) to allow you to define styles that can be re-used throughout your stylesheet. We divide them into these categories:


<Tabs items={['Global mixins', 'Layout & Spacing Mixins', 'Focus Ring Mixins']}>
  <Tab>
    ```scss copy
    $base: 16px !default;

    @function rem($size) {
      $rem: math.div($size, $base);

      @return #{$rem}rem;
    }

    @mixin truncate-title($max-lines: var(--fs-text-max-lines)) {
      display: -webkit-box;
      overflow: hidden;
      -webkit-box-orient: vertical;
      -moz-box-orient: vertical;
      -webkit-line-clamp: $max-lines;
      line-clamp: $max-lines;
      text-overflow: -o-ellipsis-lastline;
      text-overflow: ellipsis;
      white-space: normal;
    }
    ```
  </Tab>
  <Tab>
    ```scss copy
    @mixin layout-content {
      box-sizing: border-box;
      width: calc(100% - var(--fs-grid-padding) - var(--fs-grid-padding));
      max-width: var(--fs-grid-max-width);
      margin-right: auto;
      margin-left: auto;
    }

    @mixin layout-content-full {
      width: 100%;
      max-width: 100%;
    }

    @mixin layout-colored($bkg-color: var(--fs-color-neutral-bkg), $padding-top: var(--fs-spacing-5), $padding-bottom: var(--fs-spacing-7)) {
      padding-top: $padding-top;
      padding-bottom: $padding-bottom;
      background-color: $bkg-color;
    }

    @mixin section-divisor($position: "top", $padding: #{var(--fs-grid-padding)}) {
      @if $position == "top" {
        padding-top: $padding;
        border-top: var(--fs-border-width) solid var(--fs-border-color-light);

        @include media(">=notebook") { padding-top: $padding; }
      }

      @if $position == "bottom" {
        padding-bottom: $padding;
        border-bottom: var(--fs-border-width) solid var(--fs-border-color-light);

        @include media(">=notebook") { padding-bottom: $padding; }
      }
    }
    ```
  </Tab>
  <Tab>
    ```scss copy
    @mixin input-focus-ring($outline: #{var(--fs-color-focus-ring)}, $border: #{var(--fs-border-color-active)}) {
      @media not all and (min-resolution: .001dpcm) { // Target only Safari browsers
        @supports (-webkit-appearance:none) {         // Use `focus` instead of `focus-visible`
          &:hover:focus,                              // due to Safari's lack of support
          &:focus {
            border-color: $border;
            outline: none;
            box-shadow:
              0 0 0 1px var(--fs-color-body-bkg),
              0 0 0 var(--fs-border-width-thickest) $outline,
              inset 0 0 0 var(--fs-border-width) $border;
          }
        }
      }

      &:focus-visible,
      &:hover:focus-visible {
        border-color: $border;
        outline: none;
        box-shadow:
          0 0 0 1px var(--fs-color-body-bkg),
          0 0 0 var(--fs-border-width-thickest) $outline,
          inset 0 0 0 var(--fs-border-width) $border;
      }
    }

    @mixin focus-ring {
      outline: none;
      box-shadow: 0 0 0 1px var(--fs-color-body-bkg), 0 0 0 var(--fs-border-width-thickest) var(--fs-color-focus-ring);
    }

    @mixin focus-ring-visible {
      @media not all and (min-resolution: .001dpcm) { // Target only Safari browsers
        @supports (-webkit-appearance:none) {         // Use `focus` instead of `focus-visible`
          &:focus { @include focus-ring; }            // due to Safari's lack of support
        }
      }
      &:focus-visible { @include focus-ring; }
    }
    ```
  </Tab>
</Tabs>

### Custom Mixins

The process of overriding those mixins is really similar to the custom breakpoints steps:

1. Go to the `/src` folder and create the `styles` folder and inside it. You can run the following command to create them:

<Tabs items={['macOS and Linux', 'Windows']}>
  <Tab>
    ```bash copy
    mkdir src/styles
    ```
  </Tab>
  <Tab>
    ```bash copy
    mkdir src\styles
    ```
  </Tab>
</Tabs>

2. Inside the `styles` folder, create a new file called `custom-mixins.scss`

<Tabs items={['macOS and Linux', 'Windows']}>
  <Tab>
    ```bash copy
    touch src/styles/custom-mixins.scss
    ```
  </Tab>
  <Tab>
    ```bash copy
    touch src\styles\custom-mixins.scss
    ```
  </Tab>
</Tabs>

3. Choose the mixin you want to override and add it to the file:

```diff copy
- @mixin input-focus-ring($outline: #{var(--fs-color-focus-ring)}, $border: #{var(--fs-border-color-active)}) {
+ @mixin input-focus-ring($outline: #{var(--fs-color-focus-ring-danger)}, $border: #{--fs-color-danger-border}) {
  [...]
}
```

4. Run `yarn dev` in the terminal to start the development server. You should see that all components that use the `focus-ring` are now with a red shadow. Click the input to see the change.
